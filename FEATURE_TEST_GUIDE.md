# 新功能测试指南

## 已实现的新功能

### 1. 交易数据新增字段
- 邮寄日期（posting_date）- 可选
- 邮寄时间（posting_time）- 可选  
- 备注（remarks）- 可选

### 2. 设备数据新增字段
- 离库日期（checkout_date）- 当管理员更新邮寄日期时自动设置
- 回库日期（return_date）- 设置为租期结束日期

---

## 测试步骤

### 步骤1：运行数据库迁移

**目的**: 在已有的Supabase数据库中添加新列

**操作**:
1. 打开Supabase控制台
2. 进入 SQL Editor
3. 打开 `MIGRATION_ADD_FIELDS.sql` 文件的内容
4. 复制并粘贴到SQL Editor
5. 执行SQL脚本

**验证**:
- transactions 表应该有新列: posting_date, posting_time, remarks
- equipment 表应该有新列: checkout_date, return_date

---

### 步骤2：测试编辑交易功能

**目的**: 验证可以编辑邮寄日期、邮寄时间和备注

**操作**:
1. 以管理员身份登录
2. 转到 "交易管理" -> "交易列表"
3. 点击任意交易的 "编辑" 按钮
4. 应该看到新增的三个字段:
   - 邮寄日期 (date picker)
   - 邮寄时间 (time picker)
   - 备注 (textarea)
5. 填入以下数据:
   - 邮寄日期: 选择明天的日期
   - 邮寄时间: 设置为 14:30
   - 备注: "请小心搬运易碎品"
6. 点击 "保存"

**验证**:
- 模态框关闭，返回交易列表
- 刷新页面，再次打开同一交易，验证输入的数据已保存

---

### 步骤3：测试设备状态同步

**目的**: 验证编辑邮寄日期时，设备的离库日期和回库日期被正确更新

**操作**:
1. 在步骤2中编辑交易时设置了邮寄日期
2. 转到 "设备管理" -> "设备列表"
3. 查找与该交易相关的设备

**验证**:
- 该设备的 checkout_date 应该等于邮寄日期（例如: 2024-03-18）
- 该设备的 return_date 应该等于交易的 rental_end_date

**验证方法** (如果前端还没显示这些字段):
- 打开浏览器开发者工具 (F12)
- 转到 Network 标签
- 在设备列表中刷新
- 查看 `/api/admin/equipment` 响应的JSON数据
- 检查 checkout_date 和 return_date 字段

---

### 步骤4：测试交易排序和高亮功能

**目的**: 验证可以按邮寄时间排序交易，并高亮近三天的邮寄

**操作**:
1. 转到 "交易管理" -> "交易列表"
2. 在交易列表上方看到 "排序方式" 下拉菜单
3. 选择 "按邮寄时间"
4. 交易列表应该按邮寄日期从新到旧排序

**验证**:
- 带有近三天（包括今天）邮寄日期的交易应该被高亮（黄色背景）
- 没有设置邮寄日期的交易应该在列表末尾
- 交易按邮寄日期正确排序

**预期高亮**: 
- 如果今天是 2024-03-17，则设置邮寄日期为 2024-03-17、2024-03-18、2024-03-19 或 2024-03-20 的交易会被高亮

---

### 步骤5：测试设备可用日期筛选

**目的**: 验证可以按日期范围筛选可用设备

**操作**:
1. 转到 "设备管理" -> "设备列表"
2. 在设备列表上方看到日期范围筛选器:
   - "可用日期范围:" 标签
   - 起始日期输入框
   - 结束日期输入框
   - "筛选可用设备" 按钮
   - "显示全部" 按钮
3. 输入日期范围，例如:
   - 起始日期: 2024-04-01
   - 结束日期: 2024-04-10
4. 点击 "筛选可用设备"

**验证**:
- 只有在 2024-04-01 到 2024-04-10 期间可用的设备被显示
- 这意味着:
  - 设备的 in_stock = true，或
  - 设备的使用期（checkout_date 到 return_date）不与请求的日期范围重叠
- 例如:
  - 设备A: checkout_date=2024-03-15, return_date=2024-03-25（不显示，与范围重叠）
  - 设备B: checkout_date=2024-03-10, return_date=2024-03-20（不显示，与范围重叠）
  - 设备C: checkout_date=2024-04-15, return_date=2024-04-25（不显示，在范围之后）
  - 设备D: in_stock=true（显示，当前可用）

**测试场景**:
- 创建多个交易，设置不同的邮寄日期
- 验证筛选确实排除了在使用期的设备
- 点击 "显示全部" 恢复设备完整列表

### 步骤6：集成测试 - 完整工作流

**目的**: 验证整个功能流程的协调工作

**操作**及验证**:

   - 设备的 checkout_date 应该为 NULL/空

## 常见问题排除

### 问题1: 新字段没有显示在编辑模态框中
**解决**: 确保:
- 浏览器缓存已清除 (Ctrl+Shift+Delete)
- 服务器已重启
- 前端代码已更新

### 问题2: 高亮功能不工作
**解决**:
- 检查浏览器控制台是否有JavaScript错误
- 确保交易有 posting_date 值
- 验证今天日期是否正确

### 问题3: 设备日期筛选返回空结果
**解决**:
- 确保选择的日期范围有效（开始 <= 结束）
- 验证设备是否有 checkout_date 和 return_date 值
- 检查日期格式是否正确 (YYYY-MM-DD)

### 问题4: API 返回 404
**解决**:
- 确保服务器已重启
- 检查 routes/admin.js 中的路由是否正确
- 验证认证令牌是否有效

## 数据库验证SQL

运行以下SQL来验证新列是否存在：

```sql
-- 检查 transactions 表
SELECT column_name, data_type FROM information_schema.columns 
WHERE table_name = 'transactions' AND column_name IN ('posting_date', 'posting_time', 'remarks');

-- 检查 equipment 表
SELECT column_name, data_type FROM information_schema.columns 
WHERE table_name = 'equipment' AND column_name IN ('checkout_date', 'return_date');
```
